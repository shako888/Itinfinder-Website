Documentation

Welcome to the official ItinFinder documentation. ItinFinder is an AI‑powered travel planning application built with React Native and Expo, designed to generate detailed, day‑by‑day itineraries using advanced language models and real‑time web scraping.

SDK Version: 53 (React Native 0.73)

This documentation covers ItinFinder v1.0.0. For previous versions, refer to the versioned docs archive.

What is ItinFinder?

ItinFinder transforms travel ideas into structured, actionable plans. Unlike traditional itinerary apps that rely on static databases, ItinFinder leverages:

  • Groq LLaMA 3.3 70B — Ultra‑fast inference for instant generation
  • TinyFish Web Agents — Autonomous browsing for live attraction and restaurant data
  • Open‑Meteo — 7‑day weather forecasts integrated into daily plans

The app supports five languages, offline Mode, and a community‑driven explore feed. It is available as a direct APK download for Android devices.

Quick Start

Get up and running with ItinFinder in under five minutes.

Prerequisites

  • Android 8.0+ (API level 26 or higher)
  • Approximately 50 MB free storage
  • Internet connection for AI generation

Download and Install

  1. Download the latest APK from the download page.
  2. Enable "Install unknown apps" from Settings → Security.
  3. Open the downloaded ItinFinder-latest.apk file.
  4. Follow the on‑screen installation prompts.

First Generation

After installation, open ItinFinder. You will see the home screen with two generation options:

  • Simple Mode — Enter a destination and number of days. Results in ~10 seconds.
  • Deep Mode — Provide additional preferences (budget, traveler type). Live scraping takes ~30 seconds.

Each user receives 5 free daily simple Mode requests. Additional requests can be earned through rewarded advertisements or by upgrading to Pro.

Simple Mode

Simple Mode is the fastest way to generate an itinerary. It relies entirely on the Groq LLM's internal knowledge, with no external data verification.

Parameters

Parameter Type Description
destination string City or region name
days number Number of days (1‑14)
language string Output language (en, es, fr, de, ja)

Example Request

const response = await generateSimpleItinerary({
  destination: "Kyoto",
  days: 3,
  language: "en"
});

Response Structure

{
  "id": "trip_abc123",
  "destination": "Kyoto",
  "days": [
    {
      "date": "2026-04-10",
      "activities": [
        {
          "time": "09:00",
          "title": "Fushimi Inari Shrine",
          "description": "Hike through thousands of vermilion torii gates",
          "type": "attraction"
        }
      ]
    }
  ]
}

Deep Mode

Deep Mode extends Simple Mode with live web scraping via TinyFish agents. It verifies opening hours, checks restaurant availability, and surfaces current events.

Pro Feature

Deep Mode is only available to Pro users. Pro subscribers have unlimited Deep Mode usage.

Additional Parameters

Parameter Type Description
budget number Total trip budget in USD
travelerType enum solo, couple, family, group
interests array culture, food, nature, nightlife, shopping

TinyFish Agent Flow

  1. User submits destination and preferences.
  2. Groq generates a base itinerary skeleton.
  3. TinyFish agents scrape Google Maps, official attraction websites, and OpenTable.
  4. Live data is merged into the itinerary, with alternatives suggested for unavailable items.

Budget Finder

The Budget Finder Assistant helps users discover destinations that match their financial constraints. It analyzes historical flight costs, accommodation averages, and daily expenses to recommend three optimal locations.

Usage

const suggestions = await findDestinationsByBudget({
  budget: 1500,
  days: 7,
  origin: "New York"
});
// Returns array of 3 destinations with cost breakdown

Each suggestion includes estimated airfare, lodging, food, and activities, all based on real‑time pricing data.

Community Explore

Users can share their public trips and browse trending itineraries from around the world. The Explore feed is powered by Firebase Firestore and supports filtering by destination, duration, traveler type, budget and stars on itinery.

Moderation

All public trips are screened by an AI profanity filter. Reported content is reviewed within 24 hours.

Travel Journaling

During an active trip, users can log notes, photos, and ratings for each activity. Journal entries are stored locally and synced to Firebase when an internet connection is available. 

await addJournalEntry(tripId, {
  activityId: "act_456",
  note: "The torii gates were breathtaking at sunrise.",
  rating: 5,
  photos: ["uri1", "uri2"]
});

Groq Integration

ItinFinder uses the Groq Cloud API with the llama-3.3-70b-versatile model. The client is configured in lib/groq.ts.

Configuration

import Groq from 'groq-sdk';

const groq = new Groq({
  apiKey: process.env.EXPO_PUBLIC_GROQ_API_KEY,
  dangerouslyAllowBrowser: true // Required for React Native
});

Rate Limits

Groq enforces a 30 requests per minute limit per API key. ItinFinder implements client‑side throttling and queues requests when necessary.

TinyFish Agents

TinyFish is a third‑party service that provides autonomous web browsing agents. ItinFinder uses TinyFish to scrape real‑time data for Deep Mode generations.

Agent Configuration

const agent = await createAgent({
  task: "scrape_opening_hours",
  url: "https://example.com/attraction",
  extract: ["hours", "address", "phone"]
});

Agents run in isolated containers and respect robots.txt. Results are cached for 6 hours to reduce load.

Prompt Engineering

All generation prompts are version‑controlled and stored in lib/prompts/. The system prompt enforces structured JSON output with activity types, time slots, and descriptions.

System Prompt Excerpt

You are an expert travel planner. Generate a detailed day‑by‑day itinerary in valid JSON format.
Include: destination, days (array), each day with date and activities.
Each activity must have: time (HH:MM), title, description, type (attraction|food|transit).

Authentication

ItinFinder uses Firebase Authentication with email/password and anonymous sign‑in options. Auth state is persisted using AsyncStorage.

Anonymous Mode

Users can explore the app without creating an account. Trips are stored locally. To sync across devices, an account must be created.

Security Features

  • Brute‑force protection via Firebase
  • AI‑powered profanity filtering for usernames
  • Secure token refresh and revocation

Limits reset daily at 00:00 UTC. Additional requests for Simple Mode AI itinery generation can be earned by watching rewarded advertisements (up to 3 per day).

Generation API

The internal generation service exposes a unified interface for both Simple and Deep modes.

Function Signature

async function generateItinerary(
  params: GenerationParams,
  mode: 'simple' | 'deep'

Error Handling

Error Code Description
RATE_LIMITED Daily request limit exceeded
INVALID_DESTINATION Destination could not be recognized
SCRAPE_FAILED TinyFish agent encountered an error (Deep Mode only)
NETWORK_ERROR No internet connection

Trip Storage

Trip data is structured in Firestore with the following schema:

collection: trips
document: {tripId}
fields:
  - userId: string
  - destination: string
  - startDate: timestamp
  - days: array
  - isPublic: boolean
  - createdAt: timestamp
  - updatedAt: timestamp

Local storage mirrors this schema using AsyncStorage with the key @itin/trips.

User Support & FAQ

Having trouble with the app? Here are common issues and how you can solve them instantly.

Generations & Limits

"I've reached my daily limit."

Free users get 5 AI itinerary generations every 24 hours. To get unlimited generations and live web data, you can upgrade to ItinFinder Pro in your profile.

"Why is it taking so long?"

Simple trips take ~10 seconds. Deep Mode trips take ~30-60 seconds because our AI agents are researching live web data for you. Please don't close the app while it's working!

Account & Login

"I can't sign in."

Double check that your email is spelled correctly. If you've tried too many times, the app will ask you to wait 30 minutes for security reasons.

"I forgot my password."

Tap the "Forgot Password" link on the login screen, and we'll email you a secure link to reset it.

App Features

"The AI doesn't know my city."

Try entering a larger nearby city or the country name. The AI works best with established travel destinations.

"Can I use the app offline?"

You can view all your Saved Trips while offline. However, generating new trips or using the Explore feed requires an internet connection.

Connectivity

"It says 'Network Error'."

Your phone might have lost its connection. Check your Wi-Fi or mobile data bars, and try refreshing the screen.

Technical Setup & Troubleshooting

For developers or power users looking to configure ItinFinder manually or run it in a local environment.

Environment Configuration

ItinFinder requires several API keys to function in Deep Mode. Ensure your .env file contains the following:

GROQ_API_KEY=your_key_here
TINYFISH_SERVICE_URL=https://api.tinyfish.ai
FIREBASE_CONFIG=...

Common Build Errors

"Metro Bundler failed to start"

Usually caused by a port conflict (8081). Try running npx react-native start --port 8089 or killing the existing process.

"Keystore not found"

Ensure you have generated an upload key for Android. Refer to the Official React Native Docs for signing.
